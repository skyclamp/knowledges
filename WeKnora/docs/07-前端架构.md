## 2. 应用架构

### 2.1 目录结构

```
frontend/
├── index.html              # SPA 入口 HTML，加载 /config.js 运行时配置
├── vite.config.ts          # Vite 构建配置（代理、别名）
├── package.json            # 依赖清单
├── tsconfig.json           # TypeScript 配置
├── packages/               # 本地打包依赖（xlsx）
└── src/
    ├── main.ts             # 应用入口：注册插件、挂载 App
    ├── App.vue             # 根组件：OIDC 处理、TDesign 本地化
    ├── router/
    │   └── index.ts        # 路由定义 + 路由守卫
    ├── views/              # 页面视图（按功能模块分目录）
    │   ├── auth/           # 认证（登录）
    │   ├── platform/       # 平台主框架（含侧边栏布局）
    │   ├── knowledge/      # 知识库管理
    │   ├── chat/           # 对话界面
    │   ├── creatChat/      # 新建对话/首页
    │   ├── agent/          # 智能体管理
    │   ├── organization/   # 组织/空间管理
    │   └── settings/       # 系统设置
    ├── components/         # 公共组件
    ├── stores/             # Pinia 状态管理
    ├── api/                # API 请求封装（按业务分目录）
    ├── composables/        # 组合式函数（useTheme）
    ├── hooks/              # 自定义 Hooks（useKnowledgeBase 等）
    ├── i18n/               # 国际化（locales 子目录）
    ├── types/              # TypeScript 类型定义
    ├── utils/              # 工具函数（request、security 等）
    └── assets/             # 静态资源（字体、图片、主题 CSS）
```

### 2.2 模块划分方式

前端采用 **功能模块化** 划分：
- `views/` 按业务功能分目录，目录内可再有 `components/`、`settings/` 子目录
- `api/` 按业务域分目录（`auth/`、`chat/`、`knowledge-base/`、`agent/` 等），每个目录一个 `index.ts`
- `stores/` 按领域分文件（`auth.ts`、`menu.ts`、`settings.ts` 等）
- `components/` 存放跨页面复用的公共组件

### 2.3 路由结构

```
/                               → 重定向到 /platform/knowledge-bases
/login                          → 登录页（auth: false）
/platform                       → 平台主框架（platform/index.vue）
├── /platform/knowledge-bases           → 知识库列表
├── /platform/knowledge-bases/:kbId    → 知识库详情
├── /platform/knowledge-search          → 知识搜索
├── /platform/agents                    → 智能体列表
├── /platform/creatChat                 → 全局新建对话（首页）
├── /platform/knowledge-bases/:kbId/creatChat  → 知识库关联新建对话
├── /platform/chat/:chatid              → 对话详情
├── /platform/organizations             → 组织/空间列表
├── /platform/settings                  → 系统设置
└── /platform/tenant                    → 重定向到 /platform/settings
/knowledgeBase                   → 旧路由（直接进入知识库详情）
/join                            → 接受邀请（重定向到 organizations?invite_code=...）
```

**路由守卫逻辑：**
- `requiresAuth: false`：直接放行（登录页）
- 已登录用户访问 `/login`：重定向到 `/platform/knowledge-bases`
- 未登录访问受保护路由：重定向到 `/login`

## 3. 页面功能清单

### 3.1 Auth（认证）

**`/login` — Login.vue**
- 功能：邮箱 + 密码登录；支持 OIDC 单点登录（OAuth2/OpenID Connect）；用户注册
- 主要组件：动态背景动画（SVG 节点 + 连接线）、登录表单、OIDC 登录按钮
- 关键交互：登录成功后跳转到 `/platform/knowledge-bases`；OIDC 回调处理（Base64 解码 token）

### 3.2 Platform（平台主框架）

**`/platform` — platform/index.vue**
- 功能：平台整体布局容器，包含侧边导航、全局拖拽上传遮罩、全局 Settings 模态框
- 主要组件：`<Menu>` 侧边导航、`<RouterView>` 主内容区、`<UploadMask>` 拖拽上传遮罩、`<Settings>` 设置模态框
- 关键交互：全局拖拽文件触发上传遮罩（dragenter/dragleave 计数器防抖）

### 3.3 Knowledge（知识库）

**`/platform/knowledge-bases` — KnowledgeBaseList.vue**
- 功能：知识库列表；展示"全部/我的/共享"三类知识库；上传进度面板
- 主要组件：`<ListSpaceSidebar>` 空间侧边栏，知识库卡片网格，创建按钮，未初始化状态警告横条
- 关键交互：点击卡片进入知识库详情；创建新知识库（弹出 `KnowledgeBaseEditorModal`）；拖拽上传文档

**`/platform/knowledge-bases/:kbId` — KnowledgeBase.vue**
- 功能：知识库详情页；文档列表（标签分组）；文档上传、URL导入；文件重解析；FAQ 词条管理
- 主要组件：文档卡片列表、`<FAQEntryManager>`（FAQ 模式）、`<DocContent>` 文档内容预览、标签管理器、知识库配置入口
- 关键交互：上传文件（支持拖拽 + 点击）、查看文档分块 Chunks、重新解析、删除文档、移动文档到其他知识库

**`/platform/knowledge-search` — KnowledgeSearch.vue**
- 功能：跨知识库语义搜索测试
- 主要组件：搜索输入框，检索结果列表（相关度分数、来源文档）
- 关键交互：选择知识库范围、输入查询词、展示 Top-K 检索结果

**知识库设置（KnowledgeBaseEditorModal）**
- 包含子页面：数据源设置（`DataSourceSettings`）、分块设置（`KBChunkingSettings`）、模型配置（`KBModelConfig`）、解析引擎（`KBParserSettings`）、高级设置（`KBAdvancedSettings`）、存储设置（`KBStorageSettings`）、知识图谱（`GraphSettings`）、共享设置（`KBShareSettings`）

### 3.4 Chat（对话）

**`/platform/creatChat` — creatChat.vue**
- 功能：新建对话首页；展示推荐问题；输入框发起第一条消息后自动创建 Session 并跳转
- 主要组件：`<InputField>` 多功能输入框，推荐问题网格（动画过渡效果），`<KnowledgeBaseEditorModal>`
- 关键交互：@提及知识库/文件；发送消息自动跳转到对话 `/platform/chat/:id`

**`/platform/chat/:chatid` — chat/index.vue**
- 功能：对话详情页；消息历史加载（分页）；流式接收 AI 回复；停止生成
- 主要组件：消息列表（`<usermsg>` + `<botmsg>`）、`<InputField>`、推荐问题卡片
- 关键交互：滚动加载更多历史消息；流式消息实时渲染；Agent 推理过程折叠/展开；图片预览

### 3.5 Agent（智能体）

**`/platform/agents` — AgentList.vue**
- 功能：智能体列表；展示自有/共享/内置智能体；创建、编辑、删除、分享智能体
- 主要组件：`<ListSpaceSidebar>` 空间侧边栏、智能体卡片（含内置/共享/emoji 头像）、`<AgentEditorModal>`
- 关键交互：点击卡片进入新建对话（选中该 Agent）；编辑弹窗配置 Agent 参数（模型、知识库、MCP、系统提示词）

### 3.6 Organization（组织/空间）

**`/platform/organizations` — OrganizationList.vue**
- 功能：展示我创建/加入的团队空间；创建、管理、退出空间；邀请成员（生成邀请码）；接受邀请
- 主要组件：`<OrganizationSettingsModal>`（成员管理、共享资源）、`<OrganizationEditorModal>`（创建/编辑空间）
- 关键交互：URL 参数 `invite_code` 自动触发加入弹窗；申请审批

### 3.7 Settings（系统设置）

**`/platform/settings` — Settings.vue**
- 功能：系统全局设置（Teleport 到 body，覆盖层显示）
- 设置分区：
  - **常规（General）**：语言、主题、用户信息
  - **模型（Model）**：Chat/Embedding/Rerank 模型配置、Ollama 接入
  - **智能体（Agent）**：默认 Agent 配置
  - **存储引擎（StorageEngine）**：向量数据库配置
  - **解析引擎（ParserEngine）**：文档解析引擎配置
  - **检索（Retrieval）**：检索策略、重排序
  - **MCP**：MCP 服务连接配置
  - **网络搜索（WebSearch）**：搜索引擎配置
  - **聊天记录（ChatHistory）**：记忆功能配置
  - **租户信息（TenantInfo）**：API Key 管理
  - **系统信息（SystemInfo）**：版本、状态
  - **API 信息（ApiInfo）**：接口文档入口

## 5. API 通信层

### 5.1 HTTP 客户端配置

**文件：** `src/utils/request.ts`

基于 **axios** 创建实例：

```ts
const instance = axios.create({
  baseURL: import.meta.env.VITE_IS_DOCKER ? "" : "http://localhost:8080",
  timeout: 30000,
  headers: { "Content-Type": "application/json" }
})
```

**请求拦截器：**
1. 从 `localStorage` 读取 JWT Token，附加 `Authorization: Bearer <token>` 头
2. 读取 `locale` 附加 `Accept-Language` 头（支持服务端多语言响应）
3. 读取 `weknora_selected_tenant_id`，若与默认租户不同则附加 `X-Tenant-ID` 头（跨租户访问）
4. 每个请求生成随机 `X-Request-ID`

**响应拦截器：**
- 2xx 响应：直接返回 `response.data`
- 401 错误：
  - 登录接口 401：直接返回错误（展示 toast）
  - 其他接口 401：尝试刷新 Token（`/api/v1/auth/refresh`），刷新成功后重放原请求；刷新失败则清空认证状态并跳转 `/login`
  - 防重复刷新：`isRefreshing` 标志 + 失败队列 `failedQueue`

**开发代理（vite.config.ts）：**
```
/api/* → http://localhost:8080
/files/* → http://localhost:8080
```

### 5.3 SSE 流式通信

**文件：** `src/api/chat/streame.ts`

使用 `@microsoft/fetch-event-source` 实现，封装成 `useStream()` 组合式函数：

```ts
const { output, isStreaming, isLoading, error, startStream, stopStream } = useStream()
```

**核心流程：**
1. 从 `localStorage` 读取 Token，构建请求头（Authorization、X-Tenant-ID 等）
2. 调用 `fetchEventSource` 建立 SSE 连接
3. 接收到 `data` 事件时将 chunk 推入缓冲区 `buffer[]`
4. 通过定时器（可配置 `chunkInterval`）渐进式消费缓冲区，驱动 `output` ref 更新
5. 收到 `[DONE]` 信号或连接关闭时重置流状态
6. 通过 `AbortController` 支持主动停止生成

**事件格式（SSE data）：**
- 普通文本 chunk：直接追加到 `output`
- JSON 格式事件：包含 `type` 字段（`thinking`、`tool_call`、`tool_result`、`plan_task_change` 等 Agent 中间步骤事件）
- `[DONE]`：流结束信号

### 5.4 认证 Token 管理

| 场景 | 处理方式 |
|------|---------|
| 登录成功 | `authStore.setToken()` → 写入 `weknora_token` |
| 注销 | `authStore.logout()` → 清除所有 localStorage 键 |
| 请求时 | 请求拦截器读 `weknora_token` 附加 Bearer 头 |
| Token 过期（401） | 自动刷新（`/api/v1/auth/refresh`），成功则重放请求 |
| 刷新失败 | 重定向到 `/login`，防止重复跳转的 `hasRedirectedOn401` 标志 |
| OIDC 登录 | App.vue 在挂载时检测 URL 参数 `oidc_result`，Base64 解码提取 token |

## 6. 聊天界面设计

### 6.1 组件组成

```
chat/index.vue              ← 页面容器
├── 消息列表区
│   ├── usermsg.vue         ← 用户消息气泡（含 @提及标签、图片）
│   └── botmsg.vue          ← AI 回复气泡
│       ├── docInfo.vue         ← 引用来源文档列表
│       ├── deepThink.vue       ← 深度思考过程（非 Agent 模式的 <think> 块）
│       └── AgentStreamDisplay.vue  ← Agent 推理过程展示（树形展开）
│           └── ToolResultRenderer.vue  ← 工具调用结果渲染
└── Input-field.vue         ← 多功能输入框（@提及、图片上传、模型选择）
```

### 6.2 消息渲染

**Markdown 渲染管线（botmsg.vue）：**

1. 使用 `marked` 解析 Markdown 为 Token 树
2. 自定义 `Renderer`，对特殊元素进行增强处理：
   - **代码块**：调用 `highlight.js` 语法高亮；检测 `mermaid` 语言自动渲染为流程图
   - **图片**：通过 `security.ts` 中 `createSafeImage()` 将 `src` 替换为 `data-protected-src`，防止未授权请求；实际渲染时恢复（`hydrateProtectedFileImages`）
   - **链接**：自动添加 `rel="noopener noreferrer"` 和 `target="_blank"`
3. 所有 HTML 通过 **DOMPurify** 净化（配置了严格的 `ALLOWED_TAGS`/`ALLOWED_ATTR` 白名单）
4. 使用 `v-html` 在 DOM 中渲染净化后的 HTML

**Mermaid 图表：**
- `mermaidShared.ts` 负责懒初始化 Mermaid 实例（避免重复初始化）
- 代码块渲染时检测语言为 `mermaid`，生成带 `data-mermaid-code` 属性的占位 div
- 组件 `onMounted`/`onUpdated` 后调用 Mermaid API 渲染为 SVG

### 6.3 流式消息实时显示

**两阶段渲染策略：**
1. **`isStreaming` 阶段**：通过 `useStream.output` ref 驱动，每次更新重新解析 Markdown Token；使用缓冲区 + 定时器平滑渲染（避免每个 chunk 都触发全量 DOM 更新）
2. **`is_completed` 阶段**：流结束后标记，显示操作工具栏（复制、添加到知识库）

不完整的 Markdown 元素（如未闭合的图片标签）由 `replaceIncompleteImageWithPlaceholder()` 处理，避免渲染异常。

### 6.4 Agent 推理过程展示

**`AgentStreamDisplay.vue`** — 树形可折叠展示 Agent 中间步骤：

```
▼ [Agent 图标] 正在思考... （可折叠根节点）
  ├── 💭 Thinking: ...（可展开/折叠的思考过程）
  ├── 🔧 Tool Call: search_knowledge（工具调用，含参数）
  │     → Tool Result: ...（工具返回结果）
  ├── 📋 Plan Task Change: 当前任务（计划任务变更）
  └── 🔧 Tool Call: web_search（网络搜索调用）
```

每个步骤卡片有三种状态：`action-pending`（进行中）、`action-error`（失败）、完成（默认）

工具调用结果由 `ToolResultRenderer.vue` 负责通过 `tool-results/` 子目录中的不同渲染器处理不同工具类型的结果展示。

### 8.2 组件设计模式

- **组合式 API（Composition API）**：所有组件使用 `<script setup>`，通过 `ref`/`reactive`/`computed` 管理本地状态
- **Props + Emits**：明确的 Props 定义和 Emits 声明（部分旧组件使用 `defineProps` 省略类型）

## 9. 国际化

### 9.1 支持的语言

| 语言代码 | 语言 |
|---------|------|
| `zh-CN` | 简体中文（默认） |
| `en-US` | English |
| `ko-KR` | 한국어 |
| `ru-RU` | Русский |

TDesign 组件库的内置语言包同步切换（`enUSConfig`、`zhCNConfig`、`koKRConfig`、`ruRUConfig`）。

### 9.2 国际化实现方式

**初始化（`src/i18n/index.ts`）：**
```ts
const i18n = createI18n({
  legacy: false,        // 使用 Composition API 模式
  locale: savedLocale,  // 从 localStorage 读取上次设置的语言
  fallbackLocale: 'zh-CN',
  globalInjection: true // 注入 $t 到模板
})
```

**在组件中使用：**
```ts
// 组合式 API
const { t } = useI18n()
t('knowledgeBase.title')

// 模板直接用
{{ $t('menu.agents') }}
```

**语言切换：** 通过设置页面修改 `locale` 并写入 `localStorage`；`menuStore` 监听 `i18n.global.locale.value` 变更，自动重新翻译菜单项。

### 9.3 翻译文件组织

位于 `src/i18n/locales/`，每种语言一个 TypeScript 文件，导出嵌套对象：

```
locales/
├── zh-CN.ts   # 简体中文
├── en-US.ts   # English
├── ko-KR.ts   # 한국어
└── ru-RU.ts   # Русский
```

翻译 key 按功能分组（命名空间模式），例如：
- `menu.*` — 菜单项
- `knowledgeBase.*` — 知识库相关
- `agent.*` — 智能体相关
- `chat.*` — 对话相关
- `error.*` — 错误信息
- `general.*` — 通用文本

## 10. 主题与样式

### 10.1 主题方案

支持 **亮色（light）/ 暗色（dark）/ 跟随系统（system）** 三种模式。

**实现机制（`src/composables/useTheme.ts`）：**
1. 在 `main.ts` 调用 `initTheme()` 初始化（读取 `localStorage['WeKnora_theme']`）
2. 主题模式写入 `document.documentElement` 的 `theme-mode` 属性
3. 选择 `system` 时，监听 `window.matchMedia('(prefers-color-scheme: dark)')` 媒体查询变化自动切换

```html
<html theme-mode="light">  <!-- 或 "dark" -->
```

CSS 通过属性选择器响应：

```css
:root, :root[theme-mode="light"] { --td-bg-color-page: #eee; }
:root[theme-mode="dark"]         { --td-bg-color-page: #181818; }
```

### 10.2 CSS 方案

- **CSS 变量（Custom Properties）**：所有颜色、字体、圆角、阴影均通过 `--td-*` 变量定义，亮/暗模式只需切换变量值
- **Less**：用于组件局部样式，支持嵌套和变量
- **Scoped 样式**：`<style scoped>` 防止样式污染
- **无 Tailwind**：项目未使用 Tailwind CSS，依赖 TDesign + 自定义 Less 样式

### 10.3 主题配色

品牌主色为绿色系：
- 亮色：`--td-brand-color: #07c05f`
- 暗色：`--td-brand-color: #06b04d`

### 10.4 字体规范

```css
--td-font-family: PingFang SC, Microsoft YaHei, Arial Regular;
--td-font-family-medium: PingFang SC, Microsoft YaHei, Arial Medium;
```

字号系统：12px（small）→ 14px（medium）→ 16px（large）→ 20px/24px/28px（title/headline）

### 10.5 设计规范

- **圆角**：2px（small）→ 3px（default）→ 6px（medium）→ 9px（large）→ 12px（extraLarge）
- **阴影**：三级阴影 `--td-shadow-1/2/3`，用于不同层级的卡片/弹窗
- **布局**：最大对话内容宽度 `800px`，侧边栏固定宽度，支持折叠

### 发送消息流程

```
用户输入 → InputField.vue
    ↓ emit('send-msg')
chat/index.vue.sendMsg()
    ↓
createSessions() 或使用已有 session_id
    ↓
useStream().startStream({

    ↓ fetchEventSource (SSE)
服务端推流
    ↓ onmessage → buffer[] → renderTimer
output ref 渐进更新
    ↓ v-bind 到 botmsg.vue
    ↓ marked 解析 + DOMPurify 净化 + v-html 渲染
```

### Token 刷新流程

```
请求失败 (401)
    ↓
isRefreshing === false?
  ├── Yes: POST /api/v1/auth/refresh → 新 token
  │         ↓
  │     processQueue() 重放等待队列
  └── No: 加入 failedQueue 等待
    ↓
刷新失败 → authStore.logout() → 跳转 /login
```
